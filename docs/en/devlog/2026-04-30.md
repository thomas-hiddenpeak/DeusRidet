# 2026-04-30 ŌĆö Auditus Fusion Selector

## Step 18n ŌĆö Separated-source identity evidence fusion

### Context

After the upstream-vs-C++ probe, the separator itself was no longer the main
suspect. The next target was to convert the per-variant evidence spread into a
runtime selector/fusion mechanism: use only signals visible before GT scoring,
then judge the selected hypotheses against the GT timeline.

Baseline commit: `d235f96`.

### Approach

Added `tools/auditus_separatio_fusion_selector.py`, an offline evaluator that
reads the existing C++ Qwen+Dual identity reports from the R1 and R2 separator
sweeps. The selector does not use GT fields while choosing evidence. Runtime
signals are:

- predicted speaker identity;
- speaker similarity and margin;
- separated-source RMS;
- separated-source ASR text length;
- cross-variant support for the same predicted speaker.

Before the evaluation run, the benchmark pre-run checklist was applied:
`deusridet` processes were killed, page cache was dropped, and `pgrep -x
deusridet` confirmed a clean state.

The default fusion rule keeps evidence with similarity `>= 0.35`, margin
`>= 0.05`, non-empty ASR text, and at most two predicted speakers per clip.
It reports two views:

- `support_union`: selected speaker hypotheses retain all qualifying
  cross-variant separated-source evidence.
- `single_best`: each selected speaker is judged only by its highest runtime
  score evidence, serving as a conservative lower bound.

### Result

Artifacts:

- `logs/separatio_fusion_selector_r1/fusion_selector_report.md`
- `logs/separatio_fusion_selector_r1/fusion_selector_report.json`

Top30 comparison:

| Method | Two-ID | One-ID | Timeline | Correct Decisions | Wrong | Decision Acc |
|--------|-------:|-------:|---------:|------------------:|------:|-------------:|
| baseline raw_o16000 | 3 / 30 | 21 / 30 | 24 / 49 = 0.490 | 24 | 8 | 0.750 |
| fusion single_best | 3 / 30 | 22 / 30 | 25 / 49 = 0.510 | 25 | 7 | 0.781 |
| fusion support_union | 6 / 30 | 25 / 30 | 32 / 49 = 0.653 | 31 | 4 | 0.886 |

The threshold sweep found the same headline recall at both similarity `0.30`
and `0.35` when margin stayed at `0.05`; lowering margin to `0.00` or `0.03`
kept recall but increased wrong decisions from 4 to 6. A stricter similarity
`0.40` reduced wrong decisions to 3, but timeline recall fell to 29 / 49.

### Conclusion

The useful signal is not a single better separator output. It is agreement and
complementarity across separator variants. A production path should preserve
multiple high-confidence source hypotheses per predicted speaker, then fuse
speaker identity and ASR evidence before final timeline attribution. The next
implementation step is to turn this offline selector into a runtime Auditus
fusion stage while keeping the stricter margin guard that reduced wrong
speaker decisions.

## Step 18o ŌĆö Runtime fusion shadow stage, first live smoke

### Context

The offline selector needed a runtime foothold that cannot perturb online
speaker labels. The first stage therefore had to be shadow-only: run after a
final ASR segment, emit separated-source evidence, and persist it for review
without touching `SpeakerTimeline` or `on_speaker_` decisions.

Baseline commit: `d235f96`.

### Approach

Added an opt-in runtime path guarded by `DEUSRIDET_AUDITUS_FUSION_SHADOW=1`.
When a final ASR result is produced, the ASR worker now runs the initialized
MossFormer2 separator on the same `ASRJob` PCM and sends `src1` / `src2`
through the existing ASR engine as shadow evidence. The event is emitted as an
`asr_log` payload with `stage=fusion_shadow`; the Auditus facade mirrors only
that payload into `TimelineLogger::log_fusion_shadow()`.

This keeps behavior unchanged by default and avoids changing the already full
`audio_pipeline.h` interface (499 lines, R1 hard-limit pressure). The path is
runtime-observable through WebSocket and persistent timeline JSONL, but it does
not alter speaker labels.

## Step 18p ŌĆö Runtime fusion shadow speaker evidence

### Context

The runtime shadow event had separated-source ASR text but still lacked the
identity evidence needed by the production selector. The next step was to score
each separated source against the live speaker store without mutating speaker
labels, pending pools, exemplars, or timeline decisions.

### Approach

Added a shadow-only `SeparatioOratorProbe` peer under Auditus. It owns a
dedicated CAM++ encoder and Povey fbank frontend for source embedding
extraction, then scores each source through `SpeakerVectorStore::peek_best()`.
When the runtime is in dual-encoder mode, the probe combines CAM++ with the
existing WavLM-ECAPA embedding under a shared extraction mutex. The live SAAS
WavLM paths use the same mutex, so the ASR worker cannot race the real-time
speaker pipeline's scratch buffers.

The `fusion_shadow` payload now includes `src1.speaker` and `src2.speaker`
objects with readiness, extraction status, accepted flag, reason, speaker id,
similarity, second-best similarity, margin, thresholds, and timing. The path is
still opt-in behind `DEUSRIDET_AUDITUS_FUSION_SHADOW=1` and still does not alter
`SpeakerTimeline` or final transcript labels.

## Step 18q ŌĆö Runtime fusion shadow batch telemetry

### Context

The single-clip and three-clip smoke tests proved the schema, but they could
not answer whether separated-source speaker evidence is stable enough for a
production selector. The next step was a repeatable runtime batch that enrolls
known speakers first, then replays the top risky overlap clips and reviews the
`fusion_shadow` stream against the GT timeline.

### Approach

Added `tools/auditus_runtime_shadow_batch.py` with two subcommands:

- `prepare`: builds a 16 kHz mono batch WAV from clean GT reference segments,
  silence gaps, and the top risky clips from
  `logs/segment_homogeneity_clips_r3/clip_manifest.jsonl`.
- `summarize`: reads WebSocket raw events, maps runtime speaker ids to GT
  speakers from reference births, and classifies each separated source as
  `mapped_gt_hit`, `mapped_gt_wrong`, `accepted_unmapped`, or `abstain`.

The summary is telemetry only, not the primary speaker-attribution benchmark.
It uses direct runtime events to make manual review tractable, while preserving
the rule that the final benchmark score cannot be a purely scripted claim.

The first r1 batch used 8 short references plus 12 risky clips. It exposed two
issues: a summarizer bug treated `speaker_id=0` as missing, and the online
store did not reliably birth all four reference speakers from short clips. The
parser now preserves zero ids, and speaker-id mapping prefers `speaker new`
births near reference spans instead of timeline context-fill labels.

The shadow speaker margin was tightened from `0.05` to `0.055` by default
(`DEUSRIDET_AUDITUS_FUSION_SPK_MIN_MARGIN` can still override it). Offline
what-if over the captured r1 evidence showed this was the smallest change that
removed the sole borderline false accept without broadly reducing useful hits.

## Step 18r ŌĆö Model residency budget guardrail

### Context

The runtime Auditus work added a large practical caveat to the old memory
budget: the speaker stack is no longer CAM++-only when dual speaker mode is
enabled. `awaken` configures WavLM-ECAPA by default, while ASR remains
`DEUSRIDET_TEST_WS_ENABLE_ASR=1` gated for speaker-only benchmark runs and
MossFormer2 remains lazy-loaded.

### Result

Updated `.github/copilot-instructions.md` and the bilingual Machina RFC to make
model residency explicit. The production resident estimate now counts LLM,
ASR, TTS, CAM++ + WavLM-ECAPA, and small VAD/enhancement sidecars at roughly
47.0 GB before OS/CUDA overhead, leaving about 13-14 GB for KV Cache,
activations, recurrent state, long-term memory hot sets, and shadow probes.

The recorded rule is simple: before adding a large allocation, state whether it
is always-resident, env-gated, lazy, or offline-only; count the selected model
path rather than whole model directories; and do not make Tegra budget decisions
from `cudaMemGetInfo` alone.

## Step 18s ŌĆö Runtime fusion shadow arbitrium gate

### Context

The runtime shadow path had source ASR and read-only speaker evidence, but it
still lacked a conservative explanation layer. Production promotion requires
more than a high similarity score: the store must be stable, contradictions
must stay visible, and shadow evidence must remain non-authoritative until it
survives regression.

### Approach

Extended `fusion_shadow` source speaker evidence with `stable`,
`exemplar_count`, `match_count`, and the stability thresholds. The defaults are
`DEUSRIDET_AUDITUS_FUSION_STABLE_MIN_EXEMPLARS=1` and
`DEUSRIDET_AUDITUS_FUSION_STABLE_MIN_MATCHES=2`, both env-overridable.

Added a shadow-only `arbitrium` object to each `fusion_shadow` event. It never
mutates timeline labels and always reports `authoritative=false`; its actions
are explanatory candidates such as `observe_only`, `support_timeline`,
`support_single_speaker`, `propose_split`, and `record_contradiction`. A stable
source conflict with the existing timeline is now explicitly recorded as a
contradiction rather than hidden behind support language.

The batch summarizer now reports arbitrium action/reason counts plus stable
source evidence, so future tuning cycles can review the shadow gate directly.
