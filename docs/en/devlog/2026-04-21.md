### Design crystals

**Why the router became a peer Actus TU, not part of auditus_facade.**
The 545-line switch dispatches to AudioPipeline (vad/gain/speaker DB
CRUD), SpeakerTracker, *and* ConscientiStream (enables, sampling,
prompts, text injection). Forcing it into `auditus_facade` would have
dragged Conscientia/Persona headers across the R3 boundary. An Actus
verb is by construction a cross-subsystem integration point, so the
router belongs *beside* `cmd_test_ws.cpp`, not inside any single
subsystem facade. Namespace `deusridet::actus`, not
`deusridet::auditus`.

**Why connect/disconnect stayed out of Auditus (Step 7c scope cut).**
The `set_on_connect` handler broadcasts a 4096-byte consciousness
state envelope (wakefulness, KV cache metrics, persona, per-pipeline
sampling configs) plus a separate prompts envelope. None of that is
Auditus territory — it is a client-greeting surface that belongs to
a future Nexus- or Conscientia-level facade. Keeping it in 7c would
have poisoned the Auditus facade's type boundary. We pulled it out,
added Step 7e as the correct home: a peer Actus helper
`send_consciousness_hello(...)` that takes the minimum set of refs.

**Facade option A (stateless).** `auditus_facade` is a namespace of
free `install_*_callback(...)` functions. No facade class, no locks,
no new threads. AudioPipeline and WsServer continue to own their
lifetimes; the facade only wires callbacks. This preserved the
existing timeline-log ↔ WS ordering contract exactly, because the
callbacks still run on the same worker threads.
