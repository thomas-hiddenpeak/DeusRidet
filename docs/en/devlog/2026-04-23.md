# DEVLOG — 2026-04-23

## Step 10 — Actus charter restoration (5 atomic commits)

User flagged a naming regression: every TU under `src/actus/` carried
the `cmd_` prefix (a Step-3 hold-over), and many were not Actus verbs
at all — they were engine probes, kernel-timing benches, and
integration tests that happened to live in `src/actus/` only because
the Step 5 split landed every CLI subcommand there. The `cmd_` prefix
was the single most visible regression flagged in user review.

### Diagnosis

Per the philosophy: *Actus = entity acting in the world*. A command
that profiles a kernel is the developer measuring the engine, not the
entity acting. A command that round-trips a tokenizer is an integration
check, not an act. These belong in `tools/` and `tests/integration/`
respectively. The principal Actus verb of DeusRidet — the act that
brings the entity online — was hidden under the developer-flavoured
label `test-ws`. That label was honest in Step 5 (when WebSocket was
being prototyped); it has not been honest since the consciousness
stream moved into the same loop.

### Result

`src/actus/` now contains exactly six TUs:

```
src/actus/
├── actus.{h,cpp}              # shared (signal handler, usage)
├── awaken.cpp                 # principal verb: entity becomes awake
├── awaken_router.{h,cpp}      # WS text-frame command router
├── awaken_hello.{h,cpp}       # WS connect hello envelope
├── load_model.cpp             # one-shot weight load + inspect
└── load_weights.cpp           # structured weight load + validate
```

Every name now honestly describes an entity acting in the world.

### Mechanics that mattered

- Each migrated tool/test became its own `add_executable()` linking
  only the libraries it actually needs (machina + communis + CUDA).
  Previously every probe rode on `deusridet`'s full subsystem link
  list — a coupling debt invisible at the source level but real at
  link time.
- Bodies preserved verbatim; only `main()` wrappers and includes
  changed. New helper `tools/dev_main_helper.h` resolves
  `--config / --model-dir / first-positional` argument with the same
  rules `main.cpp` uses.
- `test_forward` and `test_sample` reference `g_shutdown_requested`
  (defined in `actus.cpp`); standalone TUs define a local stub so
  body code stays byte-identical.
- Canonical verification ritual now uses `./build/deusridet awaken`
  in `.github/instructions/workflow.instructions.md` and
  `.github/prompts/verify-change.prompt.md`.

Each sub-commit verified `HTTP=200 WS=101` independently.

### Lesson

Tech-dynamics — the pull toward "shortest path" — manifested here as
"every CLI verb gets a `cmd_` prefix and lives in `src/actus/`,
because that's where the split-script put them." Convenient at
generation time; obscuring at every reading thereafter. The fix is
not just renaming files but restoring the charter: each subdirectory
must mean what it claims.
