# 2026-04-27 — Step 18a: Segment Homogeneity Audit

## Context

Step 17b-A made RetroFullRing live amendments safe, but it deliberately did not
solve short-segment coverage. The next risk is not threshold tuning; it is
whether a single runtime VAD segment is speaker-homogeneous enough to carry one
speaker label. Short interjections and stance phrases must be surfaced for
human listening review before any online inheritance or segmentation change.

## Implementation

- Added `tools/segment_homogeneity_audit.py`, a read-only candidate generator.
- Inputs: timeline VAD intervals, `test_ground_truth_v1.jsonl`, coarse
  `test_ground_truth.json`, and replay artifacts from `online_replay_score.py`.
- Outputs: ranked JSONL and Markdown candidate lists under `/tmp`, including
  VAD interval, overlapping GT speakers/turns, overlapping ASR snippets, nearby
  speaker events, and human-readable risk reasons.
- The tool does not compute an accuracy metric and does not change online
  Auditus behavior.

## Step 18c-B: Shadow ASR-Boundary Split Evaluation

### Context

The ASR-merge attribution pointed toward ASR-boundary splitting, but this still
needed a negative-control check. A naive full replacement could reduce wrong
speaker propagation while destroying coverage.

### Implementation

- Added `tools/shadow_asr_boundary_split.py`, a read-only evaluator.
- Compared three variants over the same r3 replay artifacts:
  - `asr_whole`: current transcript-level speaker id.
  - `vad_split_strict`: split each ASR transcript by overlapping VAD containers;
    unpaired VAD spans abstain.
  - `vad_split_fallback`: same split, but unpaired VAD spans fall back to the
    original ASR transcript speaker id.

### Verification

Inputs: `asr=150`, `vad=257`, `speaker_events=86`, paired VAD speakers `86`.

| Variant | Micro | Macro | Coverage | Decided micro | Decided / GT |
|---------|------:|------:|---------:|--------------:|-------------:|
| asr_whole | 0.702 | 0.688 | 0.975 | 0.720 | 193/198 |
| vad_split_strict | 0.278 | 0.280 | 0.293 | 0.948 | 58/198 |
| vad_split_fallback | 0.636 | 0.637 | 0.823 | 0.773 | 163/198 |

Artifacts:

- `logs/segment_homogeneity_clips_r3/shadow_asr_split_summary.md`
- `logs/segment_homogeneity_clips_r3/shadow_asr_split_summary.json`
- `logs/segment_homogeneity_clips_r3/vad_split_strict_segments.jsonl`
- `logs/segment_homogeneity_clips_r3/vad_split_fallback_segments.jsonl`

### Result

Full replacement is not viable yet. Strict VAD-speaker splitting is highly
accurate when it decides, but it only covers 29.3% of GT segments because most
VAD spans do not have reliable speaker events. Falling back to the original ASR
speaker restores coverage, but still underperforms the current whole-transcript
assignment.

The actionable form is narrower: expose ASR split sub-spans as shadow metadata
for high-risk multi-VAD transcripts, while separately improving speaker-event
coverage. Do not replace transcript-level speaker labels globally.

## Step 18c-C: High-Risk ASR Shadow Subspan Metadata

## Step 18d: Real-Clip MossFormer2 Separation Probe

### Context

The next question was no longer theoretical: use the already exported high-risk
homogeneity clips to see whether the speech separation model is actually usable
on DeusRidet's failure cases.

### Implementation

- Added `tools/auditus_separatio_examen.cpp` and CMake target
  `auditus_separatio_examen`.
- The tool reads exported WAV clips, runs native CUDA MossFormer2 via
  `SpeechSeparator`, writes `mix/src1/src2` WAVs, and records per-clip metrics.
- Checked ClearVoice's MossFormer2 SS examples: public inference uses 16 kHz,
  a 2s decode window, 75% stride, and per-source RMS normalization. The probe
  now uses a 2s chunk with 0.5s overlap to match that stride shape.
- The two-source heuristic is deliberately conservative: the weaker raw source
  must carry at least 5% of the stronger raw source energy, and the two sources
  must not be highly correlated.
- The exported audio now has two forms: `audio/` preserves source energy balance
  for listening; `audio_official_rms/` preserves the ClearVoice-style
  per-source RMS normalization for reference.

### Verification

Input: 30 high-risk clips from `logs/segment_homogeneity_clips_r3/clips`.

| Metric | Value |
|--------|------:|
| Valid separations | 30 / 30 |
| Two-source candidates | 21 / 30 |
| Max chunk / overlap | 2.00s / 0.50s |
| Total audio | 116.30s |
| Total latency | 46126.9 ms |
| Aggregate RTF | 0.397 |

Artifacts:

- `logs/separatio_examen_r1/separatio_summary.md`
- `logs/separatio_examen_r1/separatio_results.jsonl`
- `logs/separatio_examen_r1/audio/`
- `logs/separatio_examen_r1/audio_official_rms/`

### Result

MossFormer2 is operational and fast enough for gated offline/shadow use on
these clips. The first listening export was misleading because per-source RMS
normalization made weak residuals sound as loud as real sources. The corrected
energy-preserving export is the one to audition. This still is not a speaker-ID
solution yet: some clips produce a strong second source, while others only
produce a weak residual source. The next safe step is to run speaker embedding
on separated sources as shadow metadata, not to rewrite online speaker labels.

## Step 18e: FRCRN Pre-Enhancement Before Separation

### Context

Listening review of the corrected energy-preserving MossFormer2 export was
still not satisfactory. The project had a prior memory that separation quality
improved only after denoising; local Auditus documentation already declares the
intended path as `PCM -> FRCRN -> MossFormer2 -> VAD`.

### Implementation

- Extended `tools/auditus_separatio_examen.cpp` with optional FRCRN
  pre-enhancement via `--frcrn` and `--frcrn-weights`.
- The probe now writes enhanced-input separation artifacts under a separate
  output directory, preserving both the raw mix (`*_rawmix.wav`) and the
  enhanced mix (`*_mix.wav`) before `src1/src2` outputs.
- The first attempt used the streaming 1s FRCRN allocation for whole clips and
  produced an impossible `0.5 ms` total FRCRN latency, so it was discarded. The
  offline probe now initializes FRCRN with a 10s window for the exported clips.

### Verification

Command shape after mandatory process cleanup and cache drop:

```bash
./build/auditus_separatio_examen \
  --frcrn \
  --clips-dir logs/segment_homogeneity_clips_r3/clips \
  --out-dir logs/separatio_examen_frcrn_r1 \
  --limit 30
```

Results on the same 30 high-risk clips:

| Metric | Raw SS | FRCRN -> SS |
|--------|-------:|------------:|
| Valid separations | 30 / 30 | 30 / 30 |
| Two-source candidates | 21 / 30 | 21 / 30 |
| Total audio | 116.30s | 116.30s |
| Total latency | 46126.9 ms | 52313.3 ms |
| Aggregate RTF | 0.397 | 0.450 |
| FRCRN latency | n/a | 6219.6 ms |

Structural comparison:

- Enhanced input RMS averaged 41.3% of the raw input RMS, so FRCRN materially
  changed the signal rather than acting as a no-op.
- Two-source decisions changed for two clips: rank 26 became a two-source
  candidate (`balance 0.0472 -> 0.2037`), while rank 28 stopped being one
  (`balance 0.2974 -> 0.0172`).
- The total two-source count stayed unchanged, so the result is not a clear
  automatic win by metrics alone.

Artifacts:

- `logs/separatio_examen_frcrn_r1/separatio_summary.md`
- `logs/separatio_examen_frcrn_r1/separatio_results.jsonl`
- `logs/separatio_examen_frcrn_r1/audio/`
- `logs/separatio_examen_frcrn_r1/audio_official_rms/`

### Result

The denoise hypothesis is now testable with listenable artifacts. FRCRN is
operational and adds only about 0.053 RTF on these clips, but automated proxies
do not prove a quality improvement. The next decision must come from listening
to the FRCRN export against the raw separation export, then using separated
source speaker embeddings only as shadow metadata if the audio is good enough.

## Step 18f: Fixed-Window Separator Adaptation
User listening confirmed the separator is useful, but noisy failed `src2`
outputs pointed to engineering adaptation. Two local issues were fixed:
`SpeechSeparator` no longer forwards short tail chunks directly and no longer
returns per-source RMS-amplified residuals. It now pads to full 2s windows,
stitches center regions with 0.5s overlap, and returns raw model-scale sources;
ClearVoice-style RMS is only an offline export view. Verification: build OK,
`test_overlap_separator --sep-only` OK, raw fixed-window run `30/30` valid and
`22/30` two-source at RTF `0.474`, FRCRN fixed-window run `30/30` valid and
`21/30` two-source at RTF `0.527`. Raw comparison changed rank12 `yes->no`
and rank24/rank26 `no->yes`; artifacts: `logs/separatio_examen_window_r1/`, `logs/separatio_examen_frcrn_window_r1/`.
