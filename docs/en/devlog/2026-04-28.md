# 2026-04-28 — Auditus Separatio ASR Validation

## Step 18g — Separated-source ASR + GT comparison

### Context

Thomas observed that `logs/separatio_examen_window_r1/` rank05 separates well enough by ear, so the next validation should not wait on manual listening. The right probe is to run ASR on each separated source (`src1` / `src2`) and compare the recognized text against the GT rows inside the same clip window.

### Approach

Baseline commit: `d235f96`.

Added two offline evaluator tools:

- `tools/auditus_separatio_asr.cpp`: loads Qwen3-ASR once, transcribes `mix/src1/src2` WAVs from a separation directory, and writes JSONL with text and latency fields.
- `tools/summarize_separatio_asr.py`: joins ASR JSONL to `logs/segment_homogeneity_clips_r3/clip_manifest.jsonl`, compares each source transcript to the clip GT rows, and reports whether `src1/src2` matched two distinct GT speakers.

The text scorer uses normalized Chinese character matching with a local short-phrase match path, because recovered separated sources can be very short (`专吃你` should still match the GT row containing `就是专治你...`). This is a diagnostic metric, not a final speaker attribution benchmark.

### Result

Rank05 smoke test (`logs/separatio_asr_window_rank05/`) passed the intended check:

- `src1` ASR: `是的，但是咋说呢？就是。` -> best GT speaker `朱杰`, score 0.706.
- `src2` ASR: `专吃你！` -> best GT speaker `唐云峰`, score 0.667.
- `separation_ok=True` for rank05.

Full fixed-window raw separation (`logs/separatio_asr_window_r1/`):

- ASR rows: 90 (`30 clips * mix/src1/src2`).
- Clips with `src1/src2` matching two distinct GT speakers: 8/30.
- Clips with at least one strong separated-source match: 29/30.
- Distinct-speaker matches: ranks 1, 4, 5, 6, 8, 9, 13, 14.

Full FRCRN + fixed-window separation (`logs/separatio_asr_frcrn_window_r1/`):

- ASR rows: 90.
- Clips with `src1/src2` matching two distinct GT speakers: 6/30.
- Clips with at least one strong separated-source match: 30/30.
- Distinct-speaker matches: ranks 1, 5, 6, 8, 13, 14.

### Conclusion

ASR+GT validation confirms Thomas's rank05 observation: the separated `src2` contains semantically recoverable speech from the second speaker. The effect is not isolated; raw fixed-window separation produces eight two-speaker recoveries across the top 30 high-risk clips. FRCRN improves one-source recoverability but does not improve the two-source separation count in this batch.

Online behavior remains unchanged. The next useful step is to use separated-source ASR and/or separated-source speaker embeddings as shadow metadata on high-risk VAD containers, then promote only cases with explicit acoustic evidence instead of pure temporal inheritance.

## Step 18h — Full-output ASR monitor

### Context

Thomas suggested using the separated-source ASR+GT path as a continuous monitor over all outputs, then tuning toward the most usable state. This turns the rank05 proof into a repeatable selection signal.

### Approach

Added `tools/auditus_separatio_monitor.py`, which compares multiple `asr_gt_compare.json` outputs, classifies every clip as `two_speaker`, `one_source`, or `unreliable`, and chooses the best current variant per rank. The monitor prefers energy-preserving audio views over `audio_official_rms` when scores tie, because RMS-normalized residuals are not the primary listening/runtime output.

Also extended `tools/auditus_separatio_examen.cpp` with CLI knobs for future sweeps: `--max-chunk`, `--overlap-samples`, `--two-source-balance`, and `--two-source-corr`.

### Result

The monitor now covers four variants:

Recommendation from the monitor: use `raw_window` as the primary two-speaker separation state and `frcrn_window` as the fallback one-source recovery state. `audio_official_rms` remains a diagnostic/presentation view; it did not improve aggregate two-speaker recovery.

## Step 18i — Production tuning sweep

### Context

Thomas clarified that parameter adjustment and validation should be run by the agent directly until the separator state is suitable for production use. The prior monitor provided the metric; this step turns it into an automated tuning loop.

### Approach

Added `tools/auditus_separatio_sweep.py`, which runs each candidate through:

1. `auditus_separatio_examen` separation,
2. `auditus_separatio_asr` transcription,
3. `summarize_separatio_asr.py` GT comparison,
4. `auditus_separatio_monitor.py` cross-variant selection.

Every separation/ASR invocation performs the benchmark cleanup sequence before running: kill `deusridet`, free port 8080, drop page caches, and verify no exact-name `deusridet` process remains.

### Sweep Result

Round 1 tested raw overlaps 4000 / 8000 / 12000 / 16000 and FRCRN overlaps 8000 / 12000. Round 2 searched around the winner: raw overlaps 14000 / 16000 / 18000 / 20000 / 24000 and FRCRN overlap 16000.

Best single production primary:

| Variant | Two-speaker | One-or-two | Unreliable |
|---------|------------:|-----------:|-----------:|
| raw_o16000 | 9 / 30 | 29 / 30 | 1 / 30 |

Best fallback:

| Variant | Two-speaker | One-or-two | Unreliable |
|---------|------------:|-----------:|-----------:|
| frcrn_o8000 | 6 / 30 | 30 / 30 | 0 / 30 |

Raw overlap 16000 gained rank 3, 15, and 16 as two-speaker recoveries compared with the earlier 8000-overlap default, but it lost some cases that 8000 handled. A per-rank oracle across all tested variants could reach 13/30 two-speaker clips, but that requires a runtime selector and is not promoted to online behavior yet.

### Production Change

Promoted `SpeechSeparatorConfig::overlap_samples` from `8000` to `16000`, matching the best single monitored production state. Added `DEUSRIDET_SEPARATOR_OVERLAP_SAMPLES` in `awaken` as a safe runtime override for future monitored sweeps without recompilation.

Current production stance:

- Primary separator state: raw MossFormer2, 2s window, 1s overlap.
- Fallback evidence state: FRCRN -> MossFormer2 remains useful for one-source recovery, but is not the default two-speaker separator.
- Online speaker-label replacement remains gated/shadow-only until a non-oracle runtime selector exists.

## Step 18j — Identity-first separatio monitor

### Context

Thomas reframed the acceptance signal: separated audio must be sent to both ASR and speaker ID, then judged against the GT timeline. The primary question is no longer whether the separated source text resembles a GT row, but whether the source can obtain the correct speaker identity for that GT row.

### Approach

Added a separated-source identity path:

1. `auditus_separatio_spkid` enrolls four named speaker prototypes from stable GT regions outside the reviewed high-risk clip windows, using the same dual encoder as online Auditus: CAM++ + WavLM-ECAPA.
2. Each separated `src1/src2` WAV is embedded and matched with read-only `SpeakerVectorStore::peek_best`.
3. `summarize_separatio_identity.py` joins separated-source ASR, speaker ID, and the GT row selected by ASR.
4. `auditus_separatio_identity_monitor.py` ranks parameter variants by correct speaker ID aligned to the GT timeline.

The new metric tracks:

- two-ID clips: both separated sources match distinct GT speakers and the speaker ID is correct,
- one-ID clips: at least one separated source has correct ASR+speaker-ID alignment,
- source decided accuracy / coverage,
- GT timeline recall: recovered GT rows over all GT rows in the monitored high-risk clips.

### Results

For the production candidate `raw_o16000`:

| Metric | Value |
|--------|------:|
| Two-ID clips | 3 / 30 |
| One-ID clips | 21 / 30 |
| Source decided accuracy | 0.750 |
| Source total accuracy | 0.545 |
| Source decided coverage | 0.727 |
| GT timeline recall | 24 / 49 = 0.490 |

R1 identity monitor still recommends `raw_o16000` as primary. R2 confirms it remains the best two-speaker identity state: `raw_o16000` has 3 two-ID clips; all nearby overlaps and FRCRN variants have 2 or fewer, except weaker one-source tradeoffs.

Speaker-ID acceptance threshold sweep on `raw_o16000` showed the tradeoff clearly:

| Threshold / margin | Two-ID | Timeline recall | Decided acc | Wrong | Decision |
|--------------------|-------:|----------------:|------------:|------:|----------|
| 0.30 / 0.00 | 3 | 0.551 | 0.692 | 12 | rejected: too many wrong IDs |
| 0.35 / 0.00 | 3 | 0.531 | 0.722 | 10 | rejected: recall gain costs errors |
| 0.35 / 0.03 | 3 | 0.490 | 0.750 | 8 | kept as safer default |
| 0.45 / 0.00 | 0 | 0.429 | 0.875 | 3 | rejected: over-abstains two-speaker cases |

### Production Decision

Keep separator production default at raw MossFormer2 with 16000-sample overlap. Keep speaker-ID acceptance conservative for separated sources: lower margins increase GT recall, but they also increase wrong speaker attribution, which violates the Auditus rule that abstaining is safer than misattribution.

Online speaker-label replacement remains gated/shadow-only. The next production lever should be a runtime selector or evidence fusion that can choose per-rank/per-source states without oracle GT.

## Step 18k — Low effective-ratio failure attribution

### Context

Separation is audibly and ASR-verifiably effective in several clips, but the identity-first two-ID rate remains low. Thomas asked why the effective ratio is so low if the separation path itself works.

### Approach

Added `tools/analyze_separatio_identity_failures.py`, which joins the raw_o16000 identity summary, separated-source ASR comparison, speaker-ID JSONL, separator metrics, and human review labels. The report attributes every unrecovered GT timeline row to one of three layers: missing ASR/separation recovery, speaker-ID abstain, or wrong speaker-ID attribution.

### Result

The low ratio is a funnel collapse, not a simple separator failure:

| Stage | Count |
|-------|------:|
| Multi-speaker GT clips | 16 |
| Separator two-active among multi-speaker clips | 15 |
| ASR two-speaker among multi-speaker clips | 9 |
| Speaker-ID two-ID among multi-speaker clips | 3 |

Across 49 GT timeline rows, `raw_o16000` recovers 24. The remaining 25 rows split as:

| Cause | Rows |
|-------|-----:|
| ASR/separator did not recover the GT row | 11 |
| Speaker ID abstained | 9 |
| Speaker ID chose the wrong speaker | 5 |

At the source level, separated-source speaker ID has 24 correct, 12 abstain, and 8 wrong decisions. Abstains are mostly real signal-quality gates: 6 below similarity, 4 below margin, and 2 with no usable embedding because the source RMS is too low. Wrong IDs concentrate on Tang Yunfeng: `唐云峰->朱杰` occurs 4 times and `唐云峰->石一` occurs 2 times.

The high-risk set itself is harsh: 41 labeled rows are in `音频多人`, 36 in `ASR合并多人`, 15 in `GT边界可疑`, and 7 in `关键短表态`. Many clips have enough separated speech for ASR text recovery, but the separated source is still too short, residual, or cross-contaminated for robust identity embedding. This explains why ASR-only two-speaker recovery can reach 9/30 while identity-valid two-ID recovery stays at 3/30.

### Conclusion

The next production lever should not be a looser global speaker-ID threshold: previous sweeps already showed that lowers precision. The better path is source-level evidence fusion and runtime selection: use ASR confidence, separator energy/correlation, speaker-ID margin, source RMS, and local temporal context to decide when a separated source carries enough identity evidence to override inherited labels.
