# 开发日志 — 2026-04-23

## 第 10 步 —— Actus 宪章复位（5 个原子提交）

用户指出命名回归：`src/actus/` 下每个 TU 都背着 `cmd_` 前缀
（第 3 步留下的尾巴），且其中许多根本不是 Actus 动词——它们是
引擎探针、内核计时基准、集成测试，仅仅因为第 5 步的拆分脚本
把每个 CLI 子命令都堆到了 `src/actus/`。`cmd_` 前缀是用户审查
中标出的最显眼的回归。

### 诊断

按哲学：*Actus = 在世间行动的实体*。一个给内核计时的命令
是开发者度量引擎，并非实体行动；一个跑分词器往返的命令是
集成检查，并非动作。它们应分别归属 `tools/` 和
`tests/integration/`。而 DeusRidet 的主 Actus 动词
——让实体觉醒的那个动作——一直藏在开发味的标签 `test-ws`
之下。这个标签在第 5 步是诚实的（彼时 WebSocket 在原型中），
但自意识流接入同一循环后早已不再诚实。

### 结果

`src/actus/` 现在恰好包含六个 TU：

```
src/actus/
├── actus.{h,cpp}              # 共用（信号处理、用法说明）
├── awaken.cpp                 # 主动词：实体觉醒
├── awaken_router.{h,cpp}      # WS 文本帧命令路由
├── awaken_hello.{h,cpp}       # WS 连接握手快照
├── load_model.cpp             # 一次性权重加载并检视
└── load_weights.cpp           # 结构化权重加载并校验
```

每个名字都诚实描述一个在世间行动的实体。

### 重要机制

- 每个迁移出去的 tool/test 成了自己的 `add_executable()`，
  仅链接它真正需要的库（machina + communis + CUDA）。此前
  每个探针都搭便车用 `deusridet` 的完整子系统链接列表——
  一笔在源码层面看不见、链接时却实在的耦合债。
- 函数体逐字保留；只有 `main()` 包装和 include 改了。新辅助
  `tools/dev_main_helper.h` 用与 `main.cpp` 一致的规则解析
  `--config / --model-dir / 首位置参数`。
- `test_forward` 和 `test_sample` 引用 `g_shutdown_requested`
  （定义在 `actus.cpp`）；独立 TU 在本地定义桩，保持函数体
  逐字节一致。
- 标准验证仪式现已改为 `./build/deusridet awaken`，更新
  `.github/instructions/workflow.instructions.md` 和
  `.github/prompts/verify-change.prompt.md`。

每个子提交独立验证 `HTTP=200 WS=101`。

### 教训

技术动力学——"最短路径"的牵引——在此体现为："每个 CLI 动词
都加 `cmd_` 前缀并放进 `src/actus/`，因为拆分脚本就是这么
放的"。生成时方便；此后每次阅读都被遮蔽。修复不只是改文件名，
而是复位宪章：每个子目录必须名实相符。
