# 2026-04-27 — Step 18a：Segment Homogeneity Audit

## 背景

Step 17b-A 已经把 RetroFullRing 实时修正做到了安全，但它刻意没有解决
短片段覆盖率。下一步的风险点不是继续调全局阈值，而是判断单个运行时
VAD 段是否足够“说话人同质”，能不能承载一个 speaker label。短插话和
短表态必须先被离线暴露出来，供人工听审，再决定是否改线上继承或切段。

## 实现

- 新增 `tools/segment_homogeneity_audit.py`，一个只读候选生成工具。
- 输入：timeline VAD 区间、`test_ground_truth_v1.jsonl`、粗粒度
  `test_ground_truth.json`，以及 `online_replay_score.py` 的 replay 产物。
- 输出：`/tmp` 下的排序 JSONL 与 Markdown 候选列表，包含 VAD 区间、
  重叠 GT 说话人/话轮、重叠 ASR 文本、附近 speaker 事件和可读风险原因。
- 该工具不计算准确率指标，也不改变线上 Auditus 行为。

## 验证

```bash
python3 -m py_compile tools/segment_homogeneity_audit.py
```

  --replay-dir /tmp/replay_retro_amend_r3 \

本次运行产生 167 个非零风险候选，并导出前 30 个。

这次 audit 证实：下一根杠杆应该是片段同质性，而不是继续调全局 speaker

## Step 18c-B：Shadow ASR-Boundary Split 评估

### 背景

ASR 合并归因指向了 ASR-boundary split，但它还需要一个反证检查。天真地
全量替换 speaker label，可能减少错误传播，同时毁掉覆盖率。

### 实现

- 新增 `tools/shadow_asr_boundary_split.py`，作为只读评估工具。
- 在同一组 r3 replay 产物上比较三种变体：
  - `asr_whole`：当前整条 transcript 级 speaker id。
  - `vad_split_strict`：把每条 ASR transcript 按重叠 VAD 容器切开；没有
    配对 speaker 的 VAD 子段 abstain。
  - `vad_split_fallback`：同样切分，但没有配对 speaker 的 VAD 子段回退到
    原始 ASR transcript speaker id。

### 验证

输入：`asr=150`、`vad=257`、`speaker_events=86`、paired VAD speakers `86`。

| 变体 | Micro | Macro | 覆盖率 | Decided micro | Decided / GT |
|------|------:|------:|-------:|--------------:|-------------:|
| asr_whole | 0.702 | 0.688 | 0.975 | 0.720 | 193/198 |
| vad_split_strict | 0.278 | 0.280 | 0.293 | 0.948 | 58/198 |
| vad_split_fallback | 0.636 | 0.637 | 0.823 | 0.773 | 163/198 |

产物：

- `logs/segment_homogeneity_clips_r3/shadow_asr_split_summary.md`
- `logs/segment_homogeneity_clips_r3/shadow_asr_split_summary.json`
- `logs/segment_homogeneity_clips_r3/vad_split_strict_segments.jsonl`
- `logs/segment_homogeneity_clips_r3/vad_split_fallback_segments.jsonl`

### 结果

全量替换现在不可行。严格 VAD-speaker 切分在有判定时非常准，但因为大多数
VAD 子段没有可靠 speaker event，只覆盖 29.3% 的 GT 段。回退到原 ASR
speaker 能恢复一部分覆盖率，但仍低于当前整条 transcript 级归属。

可执行形态应该更窄：先把 ASR split 子区间作为高风险多 VAD transcript 的
shadow metadata 暴露出来，同时单独提升 speaker-event 覆盖率。不要全局替换
transcript 级 speaker label。

## Step 18c-C：高风险 ASR Shadow Subspan Metadata

### 背景

在 negative-control split 评估之后，下一步要确认：如果 metadata 明确保持为
shadow-only，并且不声称有词级文本切分，它本身是否有用。

### 实现

- 新增 `tools/export_asr_shadow_subspans.py`。
- exporter 选择高风险 ASR transcript，与 VAD 容器求交，有配对 speaker event
  时挂上 speaker 信息，并把整条 transcript 文本保留为只读引用。
- 输出格式明确写入 `text_policy = parent_transcript_reference_only_no_word_timestamps`。

### 验证

R3 高风险导出：

| 指标 | 数量 |
|------|----:|
| ASR transcripts | 150 |
| 导出的高风险 transcripts | 31 |
| 总 subspans | 86 |
| 有 speaker 配对的 subspans | 29 |
| unknown subspans | 57 |

最强的正例是 ASR 67.80-76.70s：一条 parent transcript 跨两个 VAD 容器，
并且两个子段分别有 runtime speaker id（`0` 然后 `1`）。其他很多人工确认的
ASR merge 例子能暴露有用边界，但 speaker 子段仍然 unknown。

产物：

- `logs/segment_homogeneity_clips_r3/asr_shadow_subspans.jsonl`
- `logs/segment_homogeneity_clips_r3/asr_shadow_subspans.md`
- `logs/segment_homogeneity_clips_r3/asr_shadow_subspans_summary.json`

### 结果

这个 metadata 形态适合诊断和 WebUI inspection，但还不能用于自动纠错。它能
在不改变 transcript label 的前提下暴露 multi-VAD transcript 结构，以及少量
multi-runtime-speaker case。剩余阻塞点仍然是短 VAD 容器里的 speaker-event 覆盖率。

## Step 18d：真实 Clip MossFormer2 Separation Probe

### 背景

下一步不再讨论可能性：直接使用已经导出的高风险 homogeneity clips，确认
speech separation 模型在 DeusRidet 的真实失败样本上是否可用。

### 实现

- 新增 `tools/auditus_separatio_examen.cpp` 和 CMake target
  `auditus_separatio_examen`。
- 工具读取导出的 WAV clips，通过 `SpeechSeparator` 调用原生 CUDA MossFormer2，
  写出 `mix/src1/src2` WAV，并记录每个 clip 的指标。
- 对照了 ClearVoice 的 MossFormer2 SS 示例：公开推理使用 16 kHz、2s decode
  window、75% stride，以及 per-source RMS normalization。probe 现在使用 2s
  chunk 和 0.5s overlap，以贴近这个 stride 形态。
- two-source heuristic 刻意保守：较弱 raw source 至少要达到较强 raw source
  能量的 5%，且两路 source 不能高度相关。
- 导出的音频现在有两种：`audio/` 保留两路 source 的能量比例用于听审；
  `audio_official_rms/` 保留 ClearVoice 风格的 per-source RMS normalization
  作为对照。

### 验证

输入：`logs/segment_homogeneity_clips_r3/clips` 下 30 个高风险 clips。

| 指标 | 数值 |
|------|----:|
| 成功分离 | 30 / 30 |
| two-source candidates | 21 / 30 |
| Max chunk / overlap | 2.00s / 0.50s |
| 总音频长度 | 116.30s |
| 总延迟 | 46126.9 ms |
| Aggregate RTF | 0.397 |

产物：

- `logs/separatio_examen_r1/separatio_summary.md`
- `logs/separatio_examen_r1/separatio_results.jsonl`
- `logs/separatio_examen_r1/audio/`
- `logs/separatio_examen_r1/audio_official_rms/`

### 结果

MossFormer2 在这些 clips 上可运行，速度也足以支撑 gated offline/shadow use。
第一版听审导出会误导，因为 per-source RMS normalization 会把弱 residual 放大到
和真实 source 一样响。修正后的能量保留版 `audio/` 才是应该听的版本。结果仍
不是 speaker-ID 解法：有些 clip 的第二路 source 很强，另一些只是弱残留。下一步
安全路径是对 separated sources 跑 speaker embedding，作为 shadow metadata；不要
直接改写在线 speaker label。

## Step 18e：FRCRN 前级增强后再分离

### 背景

听审修正后的能量保留 MossFormer2 导出后，效果仍不够理想。项目里有过一个
记忆：分离质量可能需要先做降噪；本地 Auditus 文档也已经把目标路径写成
`PCM -> FRCRN -> MossFormer2 -> VAD`。

### 实现

- 扩展 `tools/auditus_separatio_examen.cpp`，新增可选 FRCRN 前级增强：
  `--frcrn` 和 `--frcrn-weights`。
- probe 会把增强输入后的分离产物写到单独输出目录；推荐听审目录中同时保留
  原始 mix（`*_rawmix.wav`）、增强后的 mix（`*_mix.wav`）以及 `src1/src2`。
- 第一次尝试把线上 streaming 的 1 秒 FRCRN allocation 直接用于整段 clip，
  得到了不可能的 `0.5 ms` 总 FRCRN 延迟，因此这次结果被丢弃。离线 probe
  现在用 10 秒窗口初始化 FRCRN，覆盖当前导出的 clips。

### 验证

完成强制进程清理和 drop caches 后运行：

```bash
./build/auditus_separatio_examen \
  --frcrn \
  --clips-dir logs/segment_homogeneity_clips_r3/clips \
  --out-dir logs/separatio_examen_frcrn_r1 \
  --limit 30
```

同一批 30 个高风险 clips 的结果：

| 指标 | 裸 SS | FRCRN -> SS |
|------|------:|------------:|
| 成功分离 | 30 / 30 | 30 / 30 |
| two-source candidates | 21 / 30 | 21 / 30 |
| 总音频长度 | 116.30s | 116.30s |
| 总延迟 | 46126.9 ms | 52313.3 ms |
| Aggregate RTF | 0.397 | 0.450 |
| FRCRN 延迟 | n/a | 6219.6 ms |

结构对比：

- 增强后输入 RMS 平均为原始输入 RMS 的 41.3%，说明 FRCRN 确实改变了信号，
  不是 no-op。
- 两个 clip 的 two-source 判定发生变化：rank 26 从非候选变为候选
  （`balance 0.0472 -> 0.2037`），rank 28 从候选变为非候选
  （`balance 0.2974 -> 0.0172`）。
- 总候选数仍是 21 / 30，所以仅凭自动指标不能说它已经明显胜出。

产物：

- `logs/separatio_examen_frcrn_r1/separatio_summary.md`
- `logs/separatio_examen_frcrn_r1/separatio_results.jsonl`
- `logs/separatio_examen_frcrn_r1/audio/`
- `logs/separatio_examen_frcrn_r1/audio_official_rms/`

### 结果

“先降噪再分离”的假设现在有了可听审的真实产物。FRCRN 已经跑通，在这些 clips
上只增加约 0.053 RTF，但自动 proxy 不能证明听感一定变好。下一步必须以听审
FRCRN 导出与裸分离导出的 A/B 对比为准；只有当音频足够好时，再把 separated
source speaker embedding 作为 shadow metadata 接进去，不直接改写在线标签。

## Step 18f：固定窗口 Separator 适配

听审确认 separation 模型有用，但失败样本的 `src2` 噪声指向工程适配问题。
本次修复两个本地偏差：`SpeechSeparator` 不再把短尾 chunk 直接送进模型，
也不再在核心 API 中返回 per-source RMS 放大的 residual。现在它会补到完整
2 秒 window，用 0.5 秒 overlap 只拼接中心区域，并返回 raw model-scale source；
ClearVoice 风格 RMS 只保留为离线导出视图。验证：构建通过，
`test_overlap_separator --sep-only` 通过；裸 fixed-window 运行 `30/30` 有效、
`22/30` two-source、RTF `0.474`，FRCRN fixed-window 运行 `30/30` 有效、
`21/30` two-source、RTF `0.527`。裸分离对比中 rank12 `yes->no`，
rank24/rank26 `no->yes`；产物在 `logs/separatio_examen_window_r1/` 和
`logs/separatio_examen_frcrn_window_r1/`。
