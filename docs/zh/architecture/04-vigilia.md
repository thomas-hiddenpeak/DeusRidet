# 04 — Vigilia（唤醒光谱）+ Somnium（梦境）

意识运行在一个**连续的唤醒梯度**上。没有二元的睡/醒开关。

```
唤醒等级
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 深度梦境       白日梦/空闲     聚焦            警觉
 16+ 分支       2-4 分支        1 主 decode     优先 decode
 无外部输入     低优输入        正常交互        关键词/情感触发
 记忆整合       发散联想        任务导向         立即响应
```

## Vigilia——唤醒监视器

- 跟踪滑动窗口内有效输入密度、注意力权重、关键词命中、情感强度。
- **平滑过渡**：不是离散状态切换，而是 Decode 分支数与 P/D 时间比的
  连续调整。
- **Prefill 触发的白日梦**：当 Prefill 片段激活高注意力但不需要外部
  响应时，fork 一个白日梦 Decode 分支以探索该联想。

## Somnium——梦境

延长的低唤醒状态执行真正的认知工作，不是被动空闲。

- **延长梦境（睡眠）**：长时间空闲后，升级为深度梦境，16+ 并发 Decode
  分支用于记忆整合、联想强化、创造性探索。
- **梦中整合**：`SomniumConsolidator` **仅**在低唤醒期间进行长期记忆
  维护：
  - **情节压缩**：合并相似情节记录，丢弃低重要性的。
  - **语义图维护**：加强频繁共激活的边；剪除衰减的边；通过 6 跳
    遍历发现新的实体-关系链接。
  - **重嵌入**：用当前模型周期性刷新最老的嵌入。
  - **驱逐积压**：处理在警觉状态下被驱逐但未完全整合的 KV 块
    （延迟整合）。

## 哲学注记

- 做梦不是停工，而是*系统维护自身的历史*。跳过梦境周期 = 允许长期
  记忆退化。
- 白日梦分支是不请自来的洞见的源头——不要以"专注"之名压制它们。
- Action 的工具调用必须是**异步的**——智能体不会在等待工具结果时
  停止思考。这是为什么 `Instrumenta` 与 Cogitatio 的分支模型整合而
  非阻塞意识 tick。
